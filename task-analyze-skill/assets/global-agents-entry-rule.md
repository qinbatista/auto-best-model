# Task Lifecycle

Merge this section into `~/.codex/AGENTS.md`.

Eligible text/code producers read the shared ladder and Obsidian `Projects/<key>/ModelExperience`; local model JSON stays read-only. Try Spark first: easy=low, complex=high. A zero-result operational failure uses the current 5.6 pair; published output returns now. Prompt/AI-instruction work loads `prompt-skill`; durable edits load `project-memory-skill`, recall first, then record reason/result/verification/files after Ending. Every non-ENDING_TASK_WORKER task writes an Ending start receipt and launches an independent Ending subagent. Ending Real writes the receipt-backed outcome to Obsidian. On quality/correctness failure, log failed project/module/file state before a new 5.6 repair lifecycle; re-present and use a different verifier. Parallelize isolated logs/docs. Final requires PASS or BLOCKED. No hook. Exact read-only uses one bounded rg per authoritative file, anchored to members; no route, plan, guessed names, unrelated skills, broad search, reread, full-file read, or pre-result check.

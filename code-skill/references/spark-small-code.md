# Schedule-Only Fast Source Production

Ordinary text-only code producers do not use Spark first. They execute the saved contextual quality pair. The optional fast producer generated from the current Codex catalog is reserved for disjoint read-only source branches in a cost-admitted graph. Today that schedule role resolves to `gpt-5.3-codex-spark` at `low`; replacement or removal does not break the quality ladder.

The schedule producer stays outside ordinary schema-version-2 quality nodes. Byte/context or explicit latency cost admission must first choose a graph; the dispatcher then admits Spark only on an explicitly marked one-source read-only branch. The adaptive merge still uses its contextual quality pair and may fuse one final exact-owned source audit.

For ordinary Auto, a zero-result, zero-token access/launch/transport failure may immediately try one stronger quality pair. Once any producer publishes a result, foreground fallback stops. Ending correctness/quality evidence is written to the matching Obsidian project/task/module/file/symbol/code context. `code-skill` ownership remains mandatory.

A model label or cached capability is not execution proof. Require one route receipt containing every attempt and its sanitized `model_learning_context`. Start Ending with `--producer-receipt`; the terminal lifecycle event records the outcome automatically. Old local `model_experience.json` remains legacy read-only. Present the completed result first; Ending Real runs independently and a different Ending verifier confirms any repair.
